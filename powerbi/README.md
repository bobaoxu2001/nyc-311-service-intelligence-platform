# Power BI Import Guide

This project exports Power BI-ready CSV tables to `outputs/sample_dashboard_data/`.

## Recommended Import Steps

1. Run the local pipeline:

   ```bash
   python src/ingest_311.py --limit 100000
   python src/transform_311.py
   python src/quality_checks.py
   python src/anomaly_detection.py
   python src/generate_insights.py
   ```

2. In Power BI Desktop, choose **Get Data > Text/CSV** and import these files:

   - `fact_service_requests.csv`
   - `dim_date.csv`
   - `dim_agency.csv`
   - `dim_borough.csv`
   - `dim_complaint_type.csv`
   - `dim_location.csv`
   - `daily_request_kpis.csv`
   - `monthly_request_kpis.csv`
   - `agency_performance_kpis.csv`
   - `borough_service_kpis.csv`
   - `complaint_type_kpis.csv`
   - `backlog_kpis.csv`
   - `anomalies.csv`

3. Create relationships:

   - `fact_service_requests[created_date_key]` to `dim_date[date_key]`
   - `fact_service_requests[agency_key]` to `dim_agency[agency_key]`
   - `fact_service_requests[borough_key]` to `dim_borough[borough_key]`
   - `fact_service_requests[complaint_type_key]` to `dim_complaint_type[complaint_type_key]`
   - `fact_service_requests[location_key]` to `dim_location[location_key]`

4. Mark `dim_date` as the date table using `dim_date[date_day]`.

5. Add the DAX measures from `powerbi/dax_measures.md`.

## Modeling Notes

- The fact table is request-grain: one row per 311 service request.
- Aggregated KPI tables are included for fast dashboard prototyping.
- `dim_location` supports map visuals when latitude and longitude are present.
- `anomalies.csv` is generated by a transparent rolling-baseline method in Python.
